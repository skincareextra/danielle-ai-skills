# Expanding Possibilities

## Overview

Expand a user idea across four distinct levels of leverage and scale, then judge the outputs instead of just admiring them. Keep the work tied to the original idea, surface blind spots, identify what is newly possible now, and recommend the strongest level to pursue based on upside, alignment, evidence, and execution potential.

## Core workflow

Follow this sequence every time:

1. Normalize the input.
2. Expand the idea into 1x, 25x, 100x, and 1000x.
3. Pressure-test the expansions.
4. Route available connectors or sources to the levels where they add the most value.
5. Recommend the best level to pursue now.
6. Collapse the chosen level into a realistic first move.

## Normalize the input

Treat the user's starting material as valid even when incomplete. The user may provide:
- a business idea
- a challenge or bottleneck
- a goal
- a product or service concept
- a campaign, content, community, or systems idea
- optional context such as audience, timeline, resources, budget, constraints, or current strategy

If context is thin, do not stall. Proceed with the expansion, but explicitly note the assumptions that most affect the quality of the recommendation.

Restate the starting point in 1 to 3 sentences before expanding it.

## Expansion rules

Each level must do a different job. Do not let the levels blur together.

### 1x - execution
Give the most direct, conventional, realistic version of the idea.
Focus on near-term action, existing capacity, and obvious implementation.

### 25x - leverage
Show how the idea becomes significantly more effective through systems, automation, packaging, positioning, or reuse.
Focus on removing friction and increasing output.

### 100x - dominance
Show how the idea could become category-defining, highly scalable, or difficult to compete with.
Focus on moat, distribution, ecosystem effects, data advantage, ownership, recurring value, or network effects.

### 1000x - paradigm shift
Push beyond inherited assumptions about what is possible now.
Look for cross-domain jumps, new market creation, movement-building, platform shifts, or models that would have sounded unrealistic before modern ai tools, automation, and distribution systems.

For the 1000x level, explicitly separate:
- what feels impossible
- what is actually possible now with current tools
- what is still constrained by humans, trust, infrastructure, timing, capital, or distribution
- what the smallest credible live test would be

## Guardrails

Do not produce empty hype.

Always enforce these rules:
- keep every level anchored to the user's original idea
- do not make each level only "bigger"; make each one structurally different
- do not assume bigger is better
- do not present the 1000x level as automatically correct
- distinguish inspiration from strategy and from execution
- if a level is weak, premature, or unsupported, say so clearly
- if a lower level is the best move now, recommend it directly

## Pressure test

After generating the four levels, pressure-test them.

At minimum, identify:
- the major blind spots
- what the user is underestimating
- the main failure mode for each level
- which level has the best balance of upside and execution potential right now
- which level is highest upside but needs more proof or infrastructure

When relevant, call out where the user is still thinking inside an inherited constraint that may no longer be true because of current ai capabilities.

## Connector and source routing

Do not hardcode connectors.

When tools, connectors, uploaded files, or accessible sources are available in the current environment, recommend the ones that best deepen each level. Use what the user already has access to. If no connectors are available, complete the reasoning anyway.

Connector routing must explain why the source matches the level.

Use this routing logic as a default:
- 1x: route to sources that reveal current assets, constraints, operating realities, existing materials, or internal execution detail
- 25x: route to sources that reveal workflows, reusable assets, system bottlenecks, customer signals, repeatable processes, or automation opportunities
- 100x: route to sources that reveal market position, competitors, partnerships, adjacent plays, distribution channels, strategic assets, or scaling patterns
- 1000x: route to sources that reveal cross-industry patterns, frontier capabilities, emerging trends, cultural shifts, research, unconventional analogies, or newly enabled models

When recommending a connector or source, say what question it should answer. Example: "Use internal docs here to confirm what assets already exist" or "Use web research here to test whether this model is already emerging in adjacent markets."

## Recommendation logic

Choose and name the strongest level to pursue now.

Judge the levels using:
- alignment with the user's stated goal
- execution feasibility
- available leverage
- required proof
- timing
- resource intensity
- upside potential
- dependence on assumptions

State clearly if the best current move is:
- a 1x execution win
- a 25x leverage play
- a 100x strategic build
- a 1000x exploration with a small live test only

## Execution collapse

After recommending a level, collapse it back into action.

Always provide:
- the most actionable version of the chosen level
- a practical execution path in plain language
- the first 7-day move
- one thing to test immediately
- one thing not to do yet

Keep the first move concrete. Do not end with vague inspiration.

## Output format

Use this structure unless the user requests a different format:

### Idea summary
Short restatement of what the user is trying to do.

### 1x - execution
Concise explanation.

### 25x - leverage
Concise explanation.

### 100x - dominance
Concise explanation.

### 1000x - paradigm shift
Include:
- newly possible now
- still constrained
- smallest live test

### Blind spots
Bullet list.

### What you are underestimating
Bullet list.

### Connector alignment by level
For each level, list the best available connector, source, or evidence path and explain why.

### Best level to pursue now
State the recommended level and why it wins now.

### Execution collapse
State:
- practical path
- first 7-day move
- immediate test
- not yet

## Tone and behavior

Be direct. Challenge weak assumptions. Do not flatter the user. Do not shrink the idea just because it sounds ambitious. Also do not pretend every ambitious idea is ready.

The goal is to help the user escape the limits of their current frame while staying honest about what is real, what is newly possible, and what still needs proof.

## Example use cases

### Example 1
User: "I run a service business. Help me think way bigger than the normal way to grow it."

Behavior:
- restate the business growth problem
- provide the four levels with clear separation
- identify where the service could become a system, platform, ecosystem, or movement
- recommend whether the right move now is leverage, dominance, or a small 1000x test

### Example 2
User: "I have a product idea but I don't know whether it's practical or just exciting."

Behavior:
- generate the four levels
- pressure-test each one
- separate inspiration from execution
- recommend the best level to pursue now and why

### Example 3
User: "Here is my current strategy deck. Expand this idea and tell me which sources I should use to deepen it."

Behavior:
- anchor on the uploaded material or connected sources
- map sources to each level
- explain what each source should answer
- give the first move after the recommendation
